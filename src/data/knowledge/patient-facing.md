## MYCHART

### Core Purpose
MyChart is Epic's patient portal. Enables patients to view results, message providers, schedule appointments, access visit summaries, manage proxy access, and complete pre-visit questionnaires.

### Common Go-Live Issues & Fixes

**Patient Cannot Activate MyChart Account**
- Activation code method: Code must be generated by registration or sent via text/email. Walk patient through: mychart.org → Activate → enter code + DOB + zip.
- If activation code isn't working: Check that the patient has a valid email on file in Epic. Confirm the code hasn't expired (typically 90 days).
- If activation codes are systemically failing: → Tier 2.

**After Visit Summary Not Appearing in Portal**
- AVS posts to MyChart when the encounter is closed by the provider.
- If the encounter is closed and AVS is still not in MyChart: → Tier 2.
- Common issue: Provider closed the note but didn't complete the encounter close workflow.

**Messaging Not Routing to Correct Provider**
- Patient messages route based on the provider's In Basket configuration.
- If messages are going to wrong team: → Tier 2 (In Basket routing config).

**Online Scheduling Not Showing Availability**
- Cadence templates must be configured to release slots to MyChart.
- If templates exist but slots aren't showing online: → Tier 2 (Cadence/MyChart scheduling config).

**Proxy Access**
- Parent accessing child's MyChart, or caregiver accessing adult patient's account.
- Walk through: MyChart → Share My Record → Proxy invitation.
- Proxy access for adult patients requires patient consent. Cannot be set up by staff without patient authorization (HIPAA).
- If proxy access setup is failing systemically: → Tier 2.

## HEALTHY PLANET — POPULATION HEALTH

### Core Purpose
Healthy Planet manages patient registries, care gap tracking, outreach campaigns, and care management workflows for high-risk or chronic disease populations.

### Common Go-Live Issues & Fixes

**Registry Not Populating Patients**
- Registries pull patients based on clinical criteria (diagnosis codes, medications, lab values).
- If registry is empty on day 1: Data may not have populated yet (ETL lag) or criteria may be misconfigured.
- → Tier 2 (build team to verify criteria and data population).

**Care Gap Closing Documentation**
- Care gaps close when the qualifying service is documented in Epic.
- Walk providers through: If the service was done, document it in Epic correctly — the gap closes automatically.
- If gaps aren't closing after correct documentation: → Tier 2.

## CHEERS — CRM

### Core Purpose
Cheers is Epic's CRM (Customer Relationship Management) for patient outreach, campaign management, and communication preference tracking.

### Common Go-Live Issues & Fixes

**Campaign List Not Generating**
- Lists are generated from Clarity/Caboodle data. Day 1: data lag means lists may be empty.
- If list build is confirmed and still empty after ETL has run: → Tier 2.

**Communication Preference Not Saving**
- Walk staff through the Communication Preferences activity in the patient record.
- If saves are not persisting: → Tier 2.
