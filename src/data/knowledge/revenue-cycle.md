## RESOLUTE HB — HOSPITAL BILLING

### Core Purpose
Resolute HB handles facility billing: claim generation, charge capture, denial management, and accounts receivable for inpatient and outpatient hospital encounters.

### Common Go-Live Issues & Fixes

**Claim Not Generating After Discharge**
- Discharge triggers the billing process. If discharge was not completed in Epic (disposition + discharge order), no claim generates.
- Also check: Are there outstanding charges in error? Claims with charge errors go to a hold queue.
- If discharge is confirmed complete and no claim: → Tier 2.

**Charge Not Attaching to Correct Account**
- Charge must match the patient's account type (inpatient vs. outpatient, correct visit).
- Common issue: charges from ancillary departments (lab, radiology) posting to wrong account when multiple visits exist.
- → Tier 2 to correct charge routing.

**Denial Workflow**
- Denied claims land in a denial work queue. Walk billers through: Work Queue → select denial → review reason → resubmit or appeal.
- If denial work queue isn't populated when it should be: → Tier 2.

## RESOLUTE PB — PROFESSIONAL BILLING

### Core Purpose
Resolute PB handles physician/professional fee billing: charge posting from clinical encounters, claim generation, and remittance processing.

### Common Go-Live Issues & Fixes

**Professional Fee Charge Not Flowing from Encounter**
- PB charges are generated by the provider closing/signing the encounter with a charge-triggering order or procedure.
- If encounter is signed but charge not in PB work queue: → Tier 2.
- Teach providers: The encounter must be routed for billing (not just signed). Some orgs require explicit "route for billing" step.

**Provider NPI Not Attached to Claim**
- If NPI is missing, claim will reject.
- NPI must be set up in the provider record in Epic pre-go-live.
- If NPI is missing for a provider: → Tier 2 (provider record update).

## TAPESTRY — MANAGED CARE

### Core Purpose
Tapestry manages payer contracts, capitation, auth/referral requirements, and managed care plan configuration.

### Common Go-Live Issues & Fixes

**Contract Not Applying to Claim**
- Contract application is automatic based on payer/plan match. If contract isn't applying: → Tier 2 (contract configuration).

**Auth/Referral Requirement Not Triggering**
- If a service requires auth and it's not being required at scheduling: → Tier 2 (payer plan rules config).
